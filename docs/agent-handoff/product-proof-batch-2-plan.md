# Product Proof Batch 2 Plan

## Mission

Raise the homepage and Sample Report proof layer from static foundation to a clearer product-led decision system.

## Operating path

Free Scan -> Presence Report -> Choice Gap -> Repair Queue -> Control Snapshot

## Standard

- Clear in seconds.
- Strong in proof.
- Safe in claims.
- Obvious next action.

## Scope

1. Upgrade the homepage product proof module into a stronger live-feeling product object.
2. Keep public copy owner-facing, not operator-facing.
3. Preserve the category lane: AI Search Presence Repair.
4. Avoid Semrush/Yext clone behavior and generic SEO dashboard drift.
5. Keep outcome boundaries explicit: no ranking, revenue, lead, or AI-placement guarantees.

## Batch 2 implementation target

- More product proof density inside `ProductProofSystem`.
- A left-side decision path with a right-side report-style preview.
- Four stages: Presence Report, Choice Gap, Repair Queue, Control Snapshot.
- Each stage must show state, evidence, limitation, and next action.
