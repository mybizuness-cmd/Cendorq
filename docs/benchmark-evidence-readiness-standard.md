# Benchmark Evidence Readiness Standard

This standard defines how benchmark reference businesses should be prepared before they are used for Cendorq comparisons, report quality, optimization guidance, or monthly control learning.

## Principle

Benchmark intelligence must be source-backed. A benchmark should not become an approved reference just because it looks strong.

Every approved benchmark should have enough evidence to explain why it is a good comparison point for a specific category.

## Required benchmark metadata

Each benchmark reference should include:

- category
- business name
- source reference
- review date
- review owner
- approval status
- staleness status

## Required evidence types

Before approval, a benchmark should be reviewed against source evidence such as:

- website
- service page
- review pattern
- booking or contact path
- authority content when available

The default target is five source observations per benchmark.

## Approval checks

A benchmark should not be approved until the reviewer confirms:

- source evidence is present
- category fit is confirmed
- strong-business signal is confirmed
- customer path is reviewed
- private notes are separated from reusable customer-safe patterns

## Retirement checks

A benchmark should be retired or re-reviewed when there is:

- stale source evidence
- weakened trust signal
- category mismatch
- unavailable evidence
- a better benchmark replacement available

## Non-negotiables

- No benchmark without source evidence.
- No stale benchmark treated as an active reference.
- No private notes copied into customer-facing output.
- No benchmark used as proof by itself.
- No benchmark approval without category fit.
- Cendorq remains the source of truth.
