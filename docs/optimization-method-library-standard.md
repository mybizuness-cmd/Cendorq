# Optimization Method Library Standard

This standard defines how Cendorq organizes, tests, approves, and uses optimization recommendations across Free Scan, Deep Review, Build Fix, and Ongoing Control.

## Principle

Optimization recommendations must be method-backed, evidence-linked, plan-scoped, and useful. AI can help draft and compare recommendations, but it must not invent methods or promise outcomes that are not supported.

## Method library

The method library should organize approved improvement methods such as:

- clarity upgrade
- trust proof upgrade
- conversion path upgrade
- authority signal upgrade
- offer structure upgrade
- follow-up system upgrade
- monthly control upgrade

Each method should define:

- plan scope
- problem signals
- required evidence
- proof checks
- expected outcomes
- customer-safe output rules

## Plan scope

Recommendations must match the plan purchased or being reviewed.

A Free Scan can identify opportunities and next steps. A Deep Review can give deeper evidence-backed analysis. Build Fix can create implementation tasks. Ongoing Control can track monthly progress, risks, and next actions.

## Required proof

Before a recommendation is customer-facing, it should pass:

- problem identified
- evidence linked
- plan scope matched
- expected impact stated plainly
- unsupported claim blocked
- operator review completed

## Customer-safe output

Customer-facing recommendations should be plain, practical, and honest.

They should not:

- guarantee revenue
- invent authority
- exaggerate proof
- expose private notes
- imply certainty when the evidence only supports risk or opportunity

## Monthly control usage

Monthly control should use the library to check:

- what changed
- what improved
- what became stale
- what is still blocked
- what should happen next
- what evidence supports the next action

## Non-negotiables

- No optimization recommendation without evidence.
- No recommendation outside plan scope.
- No guaranteed results unless explicitly supported by verified evidence.
- No AI-only recommendation treated as approved.
- No private notes in customer-facing recommendations.
- No hidden method changes.
- Cendorq remains the source of truth.
