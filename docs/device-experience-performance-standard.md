# Device Experience and Performance Standard

Cendorq must feel intentional on mobile, desktop, and every practical device class.

## Core rule

Mobile is the main entrance. Desktop is the command room. Neither experience should feel like an afterthought.

## Device rules

- public pages must lead with the strongest action on mobile
- plan comparison must remain readable on mobile
- dashboard surfaces must keep state and next action visible
- billing, report vault, support, and notifications must remain recoverable on small screens
- desktop may show more context only when it improves decisions
- tap targets must be large enough to use comfortably
- layouts must avoid horizontal overflow
- sections must stack with clear hierarchy
- performance review should track LCP, INP, and CLS
- public sharing and metadata should not depend on client-only rendering

## Blocked patterns

- compressed desktop layout on mobile
- buried primary action
- cluttered sticky bars
- tiny tap targets
- pricing comparison that only works on desktop
- dashboard recovery that disappears on mobile
- decorative blocks that slow the page without improving decisions

## Final rule

Cendorq should feel fast, clear, stable, and commanding across devices while preserving the same truth, plan boundaries, and customer recovery paths.
