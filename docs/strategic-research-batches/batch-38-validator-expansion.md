# Cendorq Strategic Research Batch 38: Validator Expansion

Cendorq needs validators that make the category standard enforceable.

## Validator targets

- report package shape;
- evidence record visibility;
- report lineage completeness;
- customer-safe wording;
- plan routing consistency;
- PDF, dashboard, and email sync;
- visual module validity;
- entitlement state;
- delivery event completeness;
- unsupported guarantee language;
- raw internal leakage.

## Blocked conditions

- report has no evidence package;
- customer-facing finding has no confidence level;
- PDF and dashboard disagree;
- plan CTA is not evidence-led;
- unsupported competitor claim appears;
- raw intake or private scoring is exposed;
- Build Fix is marked complete without completion report;
- Ongoing Control snapshot lacks period and drift summary.

## Agent instruction

- Validators are not optional.
- If validation fails, repair the source or split the implementation batch.
- Do not lower the standard to pass a blocked update.
- A report that sounds good but fails evidence, access, or artifact consistency checks cannot be released.

## Final command

Validators turn doctrine into repeatable gates. Cendorq should only ship customer-facing reports, dashboards, PDFs, emails, and paid-work artifacts that pass the relevant contract and safety checks.
