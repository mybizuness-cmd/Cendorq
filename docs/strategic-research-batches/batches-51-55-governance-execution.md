## Batch 52: Implementation Roadmap by Risk

Implementation should be grouped by risk level.

Low-risk layers:

- documentation;
- copy refinements;
- visual preview improvements;
- sample report improvements;
- static contract files;
- validators that do not touch live customer data.

Medium-risk layers:

- dashboard report vault skeleton;
- report package UI;
- PDF template scaffolds;
- email template scaffolds;
- plan-state UI;
- support/status UI.

High-risk layers:

- live customer storage;
- paid entitlement reads;
- report release automation;
- PDF/email delivery to real customers;
- provider login callbacks;
- customer-owned evidence retrieval;
- external integrations.

Agent instruction:

- Use bigger batches for coherent low-risk layers.
- Use smaller batches for customer data, billing, access, delivery, and integrations.
