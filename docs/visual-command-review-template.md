## One-command clarity

- What is the safest next command?
- Is that action visible within seconds?
- Is there one dominant action instead of several competing actions?
- Does the page reduce hesitation instead of creating more decisions?

## Trust and restraint

- Does the surface feel controlled, premium, and serious?
- Is the visual system restrained rather than loud?
- Does the copy avoid unsupported certainty?
- Does the page improve trust before asking for commitment?

## Proof sequence

- Is the proof order clear?
- Can the visitor separate score, finding, proof, limitation, next move, and repair priority?
- Is there enough context for the customer to understand what happens next?
- Is anything private, raw, or internal exposed where it should not be?

## Mobile command quality

- Is the main action still clear on mobile?
- Are cards readable without crowding?
- Does the page preserve spacing and hierarchy?
- Does the mobile flow avoid becoming a compressed desktop page?

## Dashboard command quality

- Does the dashboard show current state?
- Does it show the next action?
- Does it expose report access, support path, billing path, and control posture clearly?
- Does it avoid becoming a generic dashboard?
